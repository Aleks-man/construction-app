# Demo Checklist

Recommended screenshots:

- `docs/screenshots/login.png`
- `docs/screenshots/projects.png`
- `docs/screenshots/project-details.png`
- `docs/screenshots/my-tasks.png`

Recommended demo data:

- One active project
- Two or three stages
- Several tasks with different statuses and priorities
- At least one admin, manager and worker
- At least one assigned worker task visible on the My Tasks page

Recommended review flow:

1. Start backend and frontend locally.
2. Sign in with the seeded admin account.
3. Create or open a demo project.
4. Confirm project member management works.
5. Confirm task creation, editing, deletion and status updates work.
6. Sign in as a worker and verify that only assigned task status updates are available.
